# M14: Local/Global Macros, Scalars, and Loops

## Context

Dodo translates Stata `.do` commands to SQL via a CTE chain. Currently it handles 42 commands but has no support for macros or loops (M14 in PLAN.md). These are essential for real-world .do files that use `local`/`global` for parameterization and `foreach`/`forvalues` for repetitive operations.

## Scope

### Implementing
- `local name value` / `local name = expr` / `local name "string"`
- `global name value` / `global name = expr` / `global name "string"`
- `` `name' `` expansion (locals), `$name` / `${name}` expansion (globals)
- `local ++name` / `local --name`
- `macro drop name [name...]` / `macro drop _all`
- `scalar [define] name = expr` / `scalar list` / `scalar drop`
- `scalar(name)` in expressions
- `foreach lname in list { body }`
- `foreach lname of local macname { body }`
- `foreach lname of global macname { body }`
- `foreach lname of numlist spec { body }`
- `forvalues lname = #1(#d)#2 { body }` / `forvalues lname = #1/#2 { body }`
- `display "text"` / `display expr`

Also implementing:
- `tempvar lclname [...]` ‚Äî assigns unique column names to local macros; columns with these names are dropped when the do-file/scope ends
- `tempname lclname [...]` ‚Äî assigns unique scalar/macro names to local macros; cleaned up at scope end
- Macro functions (via `` `=expr' `` or `:macro_fcn` syntax):
  - `:variable label varname` ‚Äî returns variable label from `state.variable_labels`
  - `:value label varname` ‚Äî returns value label name from `state.column_labels`
  - `:label labelname # [#_2]` ‚Äî returns label text for a value from `state.value_label_defs`
  - `:word count string` ‚Äî returns number of space-separated tokens
  - `:word # of string` ‚Äî returns the #th token
  - `:type varname` ‚Äî returns storage type (maps to DuckDB types)
  - `:display fmt expr` ‚Äî returns formatted result

### NOT implementing
- `macval()`, `macro shift`, `macro dir`, `macro list`
- `:subinstr`, `:piece`, `:strlen`, `:permname`, `:adosubdir`, `:dir`, `:sysdir`, `:environment`
- `:constraint`, `:char`, `:sortedby`, `:data label`, `:format`
- `:properties`, `:results`, matrix-related macro functions
- `:tsnorm`, `:copy`, `:list` (macro list directives)
- `foreach of varlist` / `foreach of newlist` (need column existence checks at parse time)

## Architecture

### Key Insight
Macros and scalars are **text substitution** ‚Äî they operate BEFORE the existing command pipeline. Loops are **block accumulation + repeated execution**.

### Processing Pipeline (new steps marked with *)

```
1. Read line
2. Handle comments, /// continuation
3. * ExpandMacros() ‚Äî substitute `name', $name, scalar(name)
4. * Detect foreach/forvalues ‚Üí accumulate brace block, execute iterations
5. IsDodoCommand() ‚Üí recognize command
6. TokenizeCommand() ‚Üí parse structure
7. ProcessCommand() ‚Üí generate SQL
```

## Implementation Plan

### Phase 1: State + Macro Expansion

**`src/core/dodo_core.hpp`** ‚Äî Add to `DodoState`:
```cpp
std::unordered_map<std::string, std::string> local_macros;
std::unordered_map<std::string, std::string> global_macros;
std::unordered_map<std::string, std::string> scalars;

// Tempvar/tempname tracking
std::vector<std::string> tempvar_columns;    // columns to DROP at scope end
std::vector<std::string> tempname_names;     // scalar/macro names to drop at scope end
int temp_counter = 0;                        // unique name generator: __dodo_tmp_0, __dodo_tmp_1, ...
```
Clear these in `Clear()` and `ClearAll()`.

**`src/core/dodo_core.cpp`** ‚Äî New functions:
- `ExpandMacros(text, state)` (~120 lines): scan for `` `name' ``, `$name`, `${name}`, `scalar(name)`. Also handles:
  - `` `=expr' `` ‚Äî evaluate expression inline (same as `local` with `=`)
  - `` `:macro_fcn' `` ‚Äî evaluate macro function (`:variable label`, `:word count`, etc.)
  - Iterate until no more expansions (depth limit 50).
- `EvaluateSimpleExpr(expr)` (~120 lines): recursive-descent evaluator for `+`, `-`, `*`, `/`, parens, `int()`, `round()`, `ceil()`, `floor()` on numeric literals. For `local x = 2+3` ‚Üí stores `"5"`.
- `EvaluateMacroFunction(func, state)` (~80 lines): handles `:variable label varname`, `:value label varname`, `:label name #`, `:word count string`, `:word # of string`, `:type varname`, `:display fmt expr`. Reads from existing `DodoState` maps (variable_labels, value_label_defs, column_labels).

**`src/core/dodo_core.cpp`** ‚Äî Add to `DODO_COMMANDS`:
```
"local", "global", "scalar", "macro", "display", "foreach", "forvalues", "tempvar", "tempname"
```

Add `local`, `global`, `scalar`, `macro`, `foreach`, `forvalues`, `tempvar`, `tempname` to `IsTransformationCommand()`.

**`src/core/dodo_core.cpp`** ‚Äî Command handlers in `ProcessCommand()`:
- `local`: parse `++name`/`--name`, `name = expr` (evaluate), `name value` (literal), `name "string"`. Store in `state.local_macros`.
- `global`: same logic, store in `state.global_macros`.
- `macro`: parse `drop name...` or `drop _all`. Remove from maps.
- `scalar`: parse `[define] name = expr` (evaluate+store), `list` (return SQL table), `drop` (remove).
- `tempvar lclname [...]`: generate unique names like `__dodo_tmp_0`, store in `state.local_macros[lclname]`, track in `state.tempvar_columns` for cleanup.
- `tempname lclname [...]`: same but track in `state.tempname_names` for scalar/macro cleanup.
- `display`: return `SELECT 'expanded_text' AS display`.

### Phase 2: Refactor Line Processing + Loops

**Eliminate duplication** between `ProcessDoFile()` and `process_stream()`. Extract shared `ProcessLines()`:

```cpp
using LineReader = std::function<bool(std::string &line)>;
std::vector<std::string> ProcessLines(LineReader reader, DodoState &state, bool skip_terminal);
```

Both `ProcessDoFile()` and `process_stream()` become thin wrappers providing a `LineReader`.

**Brace block accumulation** ‚Äî `AccumulateBraceBlock(LineReader)`:
- Called when `{` detected at end of a foreach/forvalues line
- Reads lines until matching `}` (tracking brace depth for nested loops)
- Returns vector of body line strings
- Handles comments and `///` continuation within the block

**Loop execution** in `ProcessLines()`:
1. After macro expansion, detect `foreach`/`forvalues` prefix
2. Parse loop header (variable name, list type, list spec)
3. Accumulate body via `AccumulateBraceBlock()`
4. For each value: set `state.local_macros[lname]`, process each body line through the full pipeline (expand macros, dispatch command)
5. After loop: keep lname set to last value (Stata behavior)

**`ParseNumlist(spec)`** (~60 lines):
- `1/5` ‚Üí 1 2 3 4 5
- `1(2)10` ‚Üí 1 3 5 7 9
- Space-separated: `1 5 10` ‚Üí 1 5 10
- Combinations: `1/3 10(5)25`

### Phase 3: REPL Integration

**`src/extension/dodo_extension.cpp`** ‚Äî In `dodo_parser_override()`:
- Apply `ExpandMacros()` to each statement before dispatch
- For loop commands: construct a `LineReader` from the `;`-split statement array. Loops in REPL use `;` as line separator within braces: `foreach x in a b c { gen y = 1 ; gen z = 2 }`
- Macros/scalars persist across REPL invocations via `DodoStateInfo`

### Phase 4: CLI (dodoc.cpp)

- Replace `process_stream()` body with call to `ProcessLines()` using istream `LineReader`
- ~20 lines net change (mostly deletion of duplicated code)

## Key Design Decisions

1. **Macro expansion before command recognition** ‚Äî ensures `` keep `varlist' `` works when `varlist` is a macro containing column names.

2. **Simple built-in expression evaluator** for `local x = expr` ‚Äî no DuckDB connection at parse time. Handles arithmetic on numeric literals. String values stored verbatim.

3. **Each loop iteration adds CTE steps normally** ‚Äî `foreach x in a b { generate y_`x' = 1 }` adds 2 steps. This is correct: each generate transforms the dataset.

4. **`scalar(name)` expanded in `ExpandMacros()`** as text substitution ‚Äî no changes needed to `TranslateExpression()`.

5. **REPL loops via `;` within braces** ‚Äî natural fit since REPL already splits on `;`.
